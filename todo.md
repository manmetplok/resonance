# Code review findings — 2026-06-10

Full-codebase review: audio path (engine + plugin DSP), separation of concerns,
KISS / lines of code. Each item was verified against the code before listing.

## Critical — audio correctness

- [x] **Recording drain corrupts channel alignment for non-power-of-two channel counts** — `resonance-audio/src/recording.rs:165-177`: `drain_ring_to_buffers` pops up to 4096 samples regardless of `input_channels`; when `4096 % channels != 0` the remainder samples are consumed but never written, permanently rotating channel alignment for the rest of the take. Fix: pop into `&mut ring_scratch[..(DRAIN_SCRATCH_LEN / channels) * channels]` so each drain consumes whole frames. Extend `tests/recording_drain.rs` beyond 2 channels (e.g. 6).

## High — audio path

- [x] **Heap allocation in `process()` via rustfft default scratch** — `plugins/resonance-eq/src/analyzer.rs:163`, `plugins/resonance-mastering/src/stages/linear_phase_eq/convolver.rs:600,659,665`, `.../design.rs:758`, `plugins/resonance-ir/src/dsp.rs:160,206`: every `fft.process()` allocates a scratch Vec (up to 64 KB per 8192-pt FFT) on the audio thread. Fix: pre-allocate scratch once and call `process_with_scratch`.
- [x] **Stable sort allocates in the audio callback** — `resonance-audio/src/mixer/midi_events.rs:86`: `sort_by_key` heap-allocates for >~20 elements; up to 512 MIDI events per buffer. Fix: `sort_unstable_by_key` with a `(sample_offset, !is_note_on)` key to keep on/off pairing deterministic.
- [x] **Seek during playback leaves notes sounding forever** — `resonance-audio/src/engine/transport.rs:347-350`: `handle_seek_to` never flushes sounding instrument voices (loop seam and Stop do). Fix: invoke the same `panic_all_instrument_plugins` path on seek while playing.
- [x] **Muted instrument tracks swallow NoteOffs → stuck notes on unmute** — `resonance-audio/src/mixer/track_block.rs:72-77,90-101`: muted/solo-suppressed tracks are skipped entirely, so NoteOffs elapsing while muted are never delivered. Fix: still collect MIDI + process into scratch and discard audio, or all-notes-off on the mute transition.
- [x] **Bounce can run concurrently with live playback on the same plugin instances** — `resonance-audio/src/engine/bounce/wav.rs:75` + `bounce/render.rs:144-152`: `to_wav` resets plugins and interleaves `process()` with the live mixer, corrupting both outputs. Fix: stop the transport (or return a `BounceError`) before offline bounce.
- [x] **Mastering bypass breaks latency alignment by ~389 ms** — `plugins/resonance-mastering/src/lib.rs:106-112` + `src/chain.rs:73-84`: bypass passes the buffer untouched while `latency_samples()` still reports the full chain latency. Fix: route bypassed signal through a latency-matched delay (internal stage bypasses already do this).
- [x] **Linear-phase FIR redesign rebuilds biquads per frequency bin, on the audio thread, every block while dragging** — `plugins/resonance-mastering/src/stages/linear_phase_eq/design.rs:740-755` (+ retrigger at `linear_phase_eq/mod.rs:860-869`, `multiband/lowpass.rs:72-77`): `band.to_biquad()` is called inside the 4097-bin loop. Minimum fix: hoist `to_biquad` out of the loop; better: debounce/offload the redesign to a worker.

## Medium — audio path

- [x] **No plugin-delay compensation: host hard-codes plugin latency to 0** — `resonance-audio/src/clap_host/process.rs:111` + `clap_host/mod.rs:40-45`: the `clap.latency` extension is never queried, so lookahead plugins time-shift their track relative to others in playback and bounce. Fix: query latency at activation and delay-compensate per chain.
- [x] **Plugin-mutex contention silently drops a block's MIDI events and audio** — `resonance-audio/src/mixer/track_block.rs:106-201`, `engine/plugins.rs:114-201`: failed `try_lock` (param drag, autosave, reload) drops collected note-ons/offs and the FX block. Fix: persist events in a queue that survives a failed `try_lock`; consider crossfading the dropout.
- [x] **No gain/pan smoothing in the mixer (zipper noise)** — `resonance-audio/src/mixer/common.rs:51-63` + `master.rs:92`: track/bus/master gain and pan are per-block constants; mute is an instant cut. Fix: ramp previous→current block gain per sample (`resonance-plugin/src/smoother.rs` has the shapes).
- [x] **Monitor ring overflow permanently rotates channel alignment** — `resonance-audio/src/mixer/mod.rs:146-150`, `platform.rs:439`, `input_pipewire.rs:387-391`: partial `push_slice` and `skip` counts aren't rounded to `frame_stride`. Fix: round producer pushes and consumer skips/reads down to whole frames. Also: the skip leaves zero jitter margin (`mixer/mod.rs:146-148`) — skip to `needed + quantum * frame_stride`.
- [x] **WAV clips never validated against engine sample rate** — `resonance-audio/src/types/clip.rs:116-156` + `engine/clips.rs:163-244`: projects opened under a different PipeWire rate play clips at the wrong pitch/speed. Fix: read the fmt-chunk rate and resample or reject/flag mismatches.
- [x] **Bounce-in-place blocks the engine dispatch loop** — `resonance-audio/src/engine/thread.rs:515-526` + `bounce/clip.rs:137`: `to_audio_clip` runs synchronously, so `CancelBounce` is undeliverable and engine services stall. Fix: run on a worker thread like `to_wav_spawn`.
- [x] **Bounce range ignores the tempo map** — `resonance-audio/src/engine/bounce_common.rs:52-58` + `bounce/wav.rs:38`: MIDI clip ends use flat `samples_per_beat()` while the renderer is tempo-aware → mis-sized renders under tempo changes. Fix: use `TempoMap::tick_to_abs_sample`.
- [x] **`all_notes_off` can silently skip high keys** — `resonance-audio/src/clap_host/instance.rs:261-267`: only keys `0..remaining` get NoteOffs when `pending_notes` is near capacity, and dropped offs are never re-queued. Fix: clear matching pending note-ons first or reserve headroom for all 128 offs.
- [x] **Reverb freeze accumulates input energy without bound** — `plugins/resonance-reverb/src/dsp/chain.rs:192-200` + `dsp/fdn.rs:671-674`: frozen FDN (decay 1.0, energy-preserving Householder) keeps summing fresh input until clipping. Fix: zero the tank input while frozen, as resonance-delay already does.
- [x] **Asymmetric saturator introduces DC with no DC blocker** — `plugins/resonance-mastering/src/stages/saturator.rs:333-341`: asymmetric shaper output has nonzero mean, amplified by the following low shelf. Fix: add a DC blocker after the shaper (see also: move `resonance-amp/src/dsp/dc_blocker.rs` into `resonance-dsp`).
- [x] **Limiter true-peak group delay not compensated** — `plugins/resonance-mastering/src/stages/limiter.rs:147-153,170-176`: the ~5.5-sample polyphase detector delay misaligns the gain constraint by ~6 samples (~0.2–0.5 dB leak past the ceiling). Fix: write the envelope constraint ~6 ring positions earlier.
- [x] **NAM model sample rate ignored** — `plugins/resonance-amp/src/nam/parse.rs`: the profile's expected rate is never parsed; running a 48 kHz capture at 44.1/96 kHz shifts the amp's response. Fix: parse the field and resample around the model, or warn in the UI.
- [x] **Declared param smoothers are dead → zipper/clicks** — `plugins/resonance-compressor/src/params.rs:164-170` vs `src/dsp.rs:99-110` (makeup/mix read raw per block) and `plugins/resonance-wavetable/src/params/fx.rs:64-99` vs `dsp/render.rs:193-197` + `dsp/effects.rs:127-135` (delay-time jumps the tap). Fix: actually advance the smoothers (as resonance-eq does) or drop the misleading `.with_smoother` declarations.

## Low — audio path

- [x] Bounce vs live solo semantics differ: `bounce/render.rs:187` computes `any_solo` over all tracks; live mixer (`mixer/mod.rs:329-332`) filters to top-level tracks. Use the same predicate.
- [x] Stopped/count-in monitoring processes only the first audible monitor track (`mixer/mod.rs:179,259`) while playback processes all. Loop over all audible monitor tracks.
- [x] Live NoteOn retry can land after its NoteOff (`resonance-audio/src/engine/midi/live.rs:38-47`); route retries through the same FIFO as live events.
- [x] CLAP transport `song_pos_beats` uses flat BPM (`mixer/mod.rs:131-135`); derive beats via `TempoMap::sample_to_abs_tick` so tempo-synced plugins don't drift on ramps.
- [ ] `eprintln!` on the audio thread when the LUFS 60-min block cap hits (`resonance-metering/src/lufs/integrated.rs`, `push_block`); use a flag the UI reports.
- [ ] Delay tempo-division/sync toggle swaps `delay_samples` discontinuously (`plugins/resonance-delay/src/sync.rs` + `dsp.rs`); smooth the resolved delay-in-samples.
- [ ] Drum kit swap hard-cuts ringing voices (`plugins/resonance-drums/src/dsp/sampler.rs:118-127`); use the existing 1024-sample `trigger_release` fade.
- [ ] Disabled multiband still runs all three linear-phase crossovers (`plugins/resonance-mastering/src/stages/multiband/mod.rs:283-295`); decide deliberately and document, or cheapen the bypass to a plain delay.
- [ ] Amp tuner runs a ~750 k-MAC YIN burst in one `process()` call (`plugins/resonance-amp/src/tuner.rs:108-118`); amortize across blocks or move to a worker fed by the ring.
- [ ] EQ analyzer DC bin reads +6 dB high (`plugins/resonance-eq/src/analyzer.rs:170-178`): bin 0 should use `2/FFT_SIZE`, not the one-sided `4/FFT_SIZE`. Display-only.
- [ ] Live MIDI input quantized to `sample_offset 0` at engine-loop cadence (`resonance-audio/src/engine/thread.rs:204-226`) although `arrival` timestamps exist; convert arrival to an intra-buffer offset if tighter feel is wanted.
